# 使用指南 / Detailed Usage Guide

## 📁 支持的输入格式
本工具目前支持以下大模型导出的聊天记录 JSON 文件：
- Google Gemini 导出记录
- Anthropic Claude 导出记录
- OpenAI ChatGPT 导出记录
- 其他类似结构的 JSON（role + parts/text 结构）

## ⚙️ 配置区域详细说明

打开 `clean_memory.py`，修改顶部配置：

~~~python
INPUT_FILE = r"input_chat_history.json"      # ← 必须修改为你的实际文件路径
OUTPUT_FILE = r"compressed_memory.txt"       # 输出文件名，可自定义
USER_NAME = ""                             # 你的名字（会显示在 TXT 中）
MODEL_NAME = ""                            # AI 的名字（会显示在 TXT 中）
~~~

## 🎯 使用示例

**场景1**：长期角色扮演记忆压缩
1. 在 Gemini/Claude 中导出完整对话历史（JSON 格式）
2. 把文件放到本项目目录，命名为 `input_chat_history.json`
3. 修改配置中的 USER_NAME 和 MODEL_NAME
4. 运行 `python ai_studio_code.py`
5. 使用生成的 `compressed_memory.txt` 重新喂给模型

**场景2**：节省 Token 恢复记忆
- 压缩前：可能占用 50k~200k+ Token
- 压缩后：通常可减少 40%~70% Token 消耗（取决于思考过程占比）

## ⚠️ 常见问题

**Q：输出文件为什么还是有空行？**  
A：工具只会压缩连续的多个空行，不会删除所有换行（保留对话可读性）。

**Q：支持一次性处理多个文件吗？**  
A：当前版本暂不支持，欢迎在 CONTRIBUTING.md 中提出或提交 PR 实现批量功能。

**Q：压缩后 Token 节省效果如何计算？**  
A：可将压缩前后的 TXT 内容分别喂给模型，比较 Context Window 使用量。

## 🚀 进阶用法
- 可将输出 TXT 作为系统提示词的一部分长期使用
- 适合需要维持几个月甚至几年角色设定的深度角色扮演用户
- 可结合本地 LLM（如你之前的 Qwen2.5 部署项目）实现完全离线记忆恢复
